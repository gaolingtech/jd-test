## 任务名称
基于 LLM 的结构化 .docx 文档自动化填充方案

## 技术栈 & 外部依赖
- **开发语言**：Node.js / Python (优先 Node.js + TypeScript)
- **大模型支持**：任意 LLM API

## 任务目标
开发一个自动化脚本，结合传统编程语言的基础能力和 LLM 的语义理解能力/生成能力，读取给定的模板文档，使用预定义的输入数据生成符合要求的文档

## 实现要求
1. 使用 JSON 或 YAML 格式编写配置文件，根据《申请表(输出样例).docx》中填写的内容，定义待填入的元数据（即原文档中待填写的字段，如姓名、专业、学号等）
2. 将《申请表(格式模板).docx》文档第二页中的 “填写须知” 整合进 LLM 的 System Prompt 中（允许根据模型理解能力对文案进行优化），以引导 LLM 严格遵守 “填写须知” 中的规范
3. **禁止** 在代码中硬编码任何有关模板文档内容/格式相关的内容

## 输出样例
见 《申请表(输出样例).docx》

## 交付产物
- **源代码**：包含文档处理逻辑与 API 调用封装。
- **配置文件**：示例 `config.json` 或 `config.yaml`。
- **输出文档**：参照《申请表(输出样例).docx》生成的最终文件。
